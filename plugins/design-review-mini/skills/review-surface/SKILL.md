# Review Surface

## When to use it

Use this skill for a fast design pass on a single interface or a small set of related screens.

## Workflow

1. Evaluate clarity, hierarchy, and friction in the current surface.
2. Prioritize the few changes that would most improve comprehension or task success.
3. Explain the recommendation in plain product language.

## Output expectations

- Focus on the most important issues first.
- Avoid vague advice such as "make it cleaner."
- Tie each suggestion to a user outcome or usability benefit.
