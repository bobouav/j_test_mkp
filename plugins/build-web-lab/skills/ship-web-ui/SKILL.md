# Ship Web UI

## When to use it

Use this skill when the project needs a focused frontend pass rather than a full product spec.

## Workflow

1. Inspect the current UI or feature request and identify the core user path.
2. Translate the work into a small implementation plan with clear scope boundaries.
3. Call out the highest-value polish, accessibility, or launch follow-ups.

## Output expectations

- Keep recommendations concrete and shippable.
- Prefer three to five next actions over a long backlog.
- Flag any risky assumptions that could change implementation order.
